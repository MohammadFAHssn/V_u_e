Computed property

we can't execute a function here for example curly brackets curly brackets setName(). because for every update and every render this function will be executed. we instead use computed property in vue.createApp({}) and place definition of that function in that computed property instead of methods property and it's important that we call it by reference (we're going to use this like a data property).

Computed properties are essentially like methods with one important difference vue will be aware of their dependencies and only reexecute them if one of the dependencies changed. off course when the app is first rendered, it render initially.

thats very very important, in computed we just return something not update

you shouldn't put too much logic into your HTML code. and add computed properties

sec 6: watch method

it's just same as computed property but it just reexecute base on just one data. thats very very import that if you wanna run logic that may be also updates a data property, we use watch but in computed we just return something. Another example, would be HTTP requests which you wanna send if certain data changes or timers which you wanna set, if certain values change or stores something in local storage. this is as same as "useEffect", we do not use them directly in templates we use them when ever ...
you should name your computed properties just as you would name your data properties.

sec 7: very important point

another very important point is that if we use a function inside of a method or computed of watch and if we use "this" keyword inside that function, this point to that function not that property! we should first outside of that function store "this" that point to a property in a variable and then use that variable inside that function.
